# Refine lessons (cross-project self-improving log)

Meta-lessons about running the refine loop well, learned across projects. Domain-specific design or code
lessons belong in the relevant domain skill or style guide, not here; this log holds only the cross-project
lessons about running the loop itself. Append a dated one-liner when a run teaches something reusable.
Format: `- [YYYY-MM-DD] Observation -> Rule.`

## Lessons

- [2026-07-08] Reviewers that judged only the source HTML missed wasted space that only shows in the render. -> Always give reviewers the real rendered snapshot, not just the source; judge reality.
- [2026-07-08] The first pass fixed the obvious (text size) but left the deeper flaw (top-aligned content in tall containers) on several units. -> Expect multiple passes; after pass 1, re-observe and run a targeted pass on units still below bar. One pass is rarely enough.
- [2026-07-08] The most effective fix was to write the newly-observed failure as a sharp lesson, then re-run; agents reading the updated lesson fixed it correctly. -> Harvest the lesson BEFORE the re-pass, not after; the loop improves within a single session, not just across sessions.
- [2026-07-08] Editing one shared file with many parallel agents would have raced. Splitting into one file per unit made the whole pass conflict-free and parallel. -> Always isolate units (own file or worktree) before fanning out writers; verify the split roundtrips before touching content.
- [2026-07-10] Refining a multi-file code feature (copilot: 3 retrieval copies + 2 endpoints + drawer) with one agent per file risked contract drift. -> For code artifacts with shared contracts, write the EXACT contract (field names, ordering, caps, pinned findings per unit) into the rubric; parallel single-file writers then compose a coordinated cross-file feature safely.
- [2026-07-10] Including the Playwright verification gate script as its own refine unit meant the new behaviors (history contract, note rendering) got machine-verified in the same pass, and the extended gate caught nothing broken. -> When the artifact has a verification script, make it a unit and instruct it to cover only rubric-pinned behaviors.
- [2026-07-10] A live parallel session kept editing the same repo (same files) while 13 refine writers ran; unique-anchor surgical Edits landed cleanly, but agents wasted effort re-diagnosing the moving tree and one agent's "stale test" finding was actually the peer's in-flight change. -> Before fanning out writers on a shared working tree, check for live peer sessions (mesh/lock); if present, pin each agent to surgical Edits on unique anchors, forbid regen/build side effects, and tell agents the tree may move under them so they attribute drift correctly.
- [2026-07-10] Constraint "risky to fix = report as FINDING (netaisyta): prefix" produced a clean triage list (stale product JSON, cross-file staleness, golden fragility) without agents overstepping their unit. -> Always give refine agents an explicit escape hatch for out-of-scope or risky findings; harvest those findings in the main loop instead of letting agents guess.
- [2026-07-10] Code-refine on a geometry engine: the highest-value findings were FRAME/SIGN convention clashes between modules (eq kairė=x<0 vs plan kairė=+offset; component projection mirrored vs everything downstream), which no single-file reviewer sees. -> For multi-module code artifacts, add one lens that explicitly hunts cross-module convention seams (units, sign, frame, ordering) and give it entry points of BOTH sides of each seam.
- [2026-07-10] A reviewer's parenthetical claim about a neighbouring module's convention was wrong (verifier repeated it); trusting it would have flipped data the opposite way. The decisive move was a 10-line physical probe (project one real point through both systems). -> When findings hinge on a convention, verify with a physical probe on real data before editing; never trust prose claims about sign/frame, even verified ones.
- [2026-07-10] Arc-length resampling replaced vertex-count sampling; the first cap (2000/geom) hung the test suite via O(samples x axis vertices) projection. -> When densifying sampling in a refine fix, budget the downstream complexity (samples x per-sample cost) and time one real call before running the suite.
- [2026-07-12] Refining an autonomous-loop PLAN (20 units, one per stage): pass-1 agents each verified their stage against the real repo and injected exact paths, model keys, parity contracts, and prior-cycle gotchas, but being blind to siblings they could not check cross-unit references (§ pointers, shared contract field names, stage-N assumes stage-M DoD). -> For multi-unit plans/docs with heavy internal cross-references, run a pass-2 coherence sweep where each agent's snapshot is the FULL reassembled document (not just its own unit), with a lens that hunts dangling § references, contract drift, and sequence-assumption breaks; keep it editing only its own unit.
- [2026-07-12] Observing a server-gated SPA (password gate authing against an /api endpoint that does not exist under `vite preview`) produced 7 screenshots that were byte-identical (same md5) - all the login card, not the scenes. The identical file sizes were the tell. -> When observing a gated or API-backed artifact, mock the auth/API boundary at the browser layer (Playwright route fulfill {authed:true}) for local QA, and ALWAYS hash the snapshots to prove they are distinct before trusting that navigation succeeded.
- [2026-07-12] A de-jargon writer translated a boolean display label (PASS/FAIL -> praeina/krenta) on one widget while the raw DATA verdict token (facts.verdict = "PASS") sitting beside it in the same view was immutable, creating an in-view mix of praeina and PASS. -> When a formal result token is data (unchangeable), pin its human-facing siblings to the SAME token in the rubric; audience-friendly prose belongs in the surrounding sentence, not in a lone translated copy of the verdict.
- [2026-07-15] Mid-refine, a live peer session renamed the WHOLE working folder (moved prezentacijos/ under silotech-programa/) on the user's instruction; my cached absolute $DIR went stale though the shell CWD followed the inode, and the mesh zone glob no longer matched. -> On a shared tree with live peers, expect structural MOVES, not just concurrent edits: re-run mesh sync, re-resolve absolute paths and re-claim the zone after any move, and message peers to leave your working folder alone before fanning out writers. Keep a backup copy (slides.orig) before destructive steps.
- [2026-07-15] Two-phase deck refine worked cleanly: the orchestrator first did the STRUCTURAL pass (strip corner chrome, insert logo, resize type, fix spacing) on the split files, then fanned out one content-enrichment agent per slide with a hard "do not re-add the removed chrome / do not touch the logo furniture" constraint. -> For artifacts mixing structural chrome with content, split the work: orchestrator owns global structure/CSS in one pass, per-unit agents own content within that frame, and pin the just-removed elements in the constraints so parallel writers do not resurrect them.
- [2026-07-16] Refining a multi-file knowledge base per-unit (47 md files, one agent each) fixed local clarity but could not see CROSS-FILE issues: contradictions, term drift, promised-but-missing content in linked files. -> After any per-unit KB pass, always run a dedicated cross-file consistency audit (contradictions, terminology, link promises, newcomer path walk) as its own verify stage; also sweep for non-Latin lookalike glyphs (Cyrillic) that slip into generated LT text.
